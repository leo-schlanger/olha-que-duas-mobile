# Release v1.1.0 (Build 35)

**Data**: 2026-04-06
**versionName**: 1.1.0
**versionCode**: 35 (Android)

---

## Resumo da Release

Release focada em estabilidade, resiliencia e melhorias de UX. Adiciona protecao contra crashes, detecao de offline, melhora significativamente a secao de programacao e refatora componentes grandes para melhor manutenibilidade.

---

## Alteracoes

### Novas Funcionalidades

- **Error Boundary global**: Componente que captura erros de renderizacao e mostra UI de fallback em vez de crashar o app inteiro
- **Detecao de offline**: Novo `NetworkContext` com `expo-network` que monitoriza o estado da rede e mostra banner vermelho "Sem ligacao a internet" quando offline
- **Indicador AO VIVO na programacao**: Badge vermelho "AO VIVO" quando um programa esta no ar, calculado com base no horario de Portugal
- **Destaque do dia atual**: Programas de hoje aparecem primeiro com borda lateral e label "Hoje"
- **Filtro de programas inativos**: Programas marcados como inativos (tanto no Supabase como no fallback) sao filtrados automaticamente

### Correcoes de Bugs

- **Memory leak no radioService**: Reconexoes de rede ja nao acumulam subscriptions do nowPlayingService. Cada reconexao limpa a subscription anterior antes de criar nova
- **Inicializacao do App**: Estados de inicializacao separados (loading/ready/failed). App ja nao trata falha de inicializacao como sucesso
- **Permissao de localizacao**: Hook `useLocation` agora re-verifica permissao quando app volta ao foreground (AppState listener). Se usuario habilita localizacao nas definicoes do sistema, o app detecta automaticamente
- **Race condition no News**: Adicionado request ID tracking para prevenir que resultados obsoletos sobrescrevam dados filtrados recentes
- **Validacao de coordenadas**: Weather service e useLocation agora validam coordenadas (NaN, Infinity, fora de range) antes de fazer chamadas a API
- **Logging consistente**: Substituido `console.warn` por `logger` em i18n e config/site.ts

### Refatoracao

- **SettingsScreen**: Dividido de 1226 linhas para componentes de secao reutilizaveis (AppearanceSection, LanguageSection, RadioSection, PremiumSection, NotificationSection, AboutSection)
- **Componentes compartilhados**: Extraidos `SettingRow` e `MenuItem` para `src/components/settings/`
- **Schedule melhorado**: Layout com icones maiores (46px), fonte do nome do programa maior (15px), linha separada para dia com icone de calendario, badges de horario mais espacados
- **Pasta vazia removida**: `src/components/radio/hooks/` que estava vazia foi removida
- **Fallback config com isActive**: Campo `isActive` adicionado ao schedule no `site.ts` para permitir desativar programas mesmo sem Supabase

### Dependencias

- Adicionado: `expo-network` (detecao de estado de rede)
