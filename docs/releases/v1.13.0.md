# Release v1.13.0 (Build 59)

**Data**: 2026-04-19
**versionName**: 1.13.0
**versionCode**: 59 (Android)

---

## Resumo da Release

Redesign da secao de programacao: painel "A Tua Soundtrack do Dia" com programas especiais integrados (logos visiveis, destaque visual). Remocao da agenda semanal e notificacoes/reminders. Validacao de cache de artwork.

---

## Alteracoes

### 1. Programacao diaria com eventos integrados

O painel "A Tua Soundtrack do Dia" agora mostra os programas especiais do dia (Nutricao, Motivar, etc.) integrados nos slots horarios com:
- Logo do programa (32x32px) via expo-image com fallback para MaterialCommunityIcons
- Destaque visual: fundo secondary/10, borda secondary/25, texto bold secondary
- Badge de duracao em cada slot (calculada automaticamente)
- Funcao `mergeTodayPrograms()` portada do site web — injeta programas semanais nos periodos corretos

### 2. Remocao da agenda semanal

- `ScheduleSection` removida do RadioPlayer (ficheiro mantido para futuro)
- `RemindersBottomSheet` removida
- `useNotifications` e toda a logica de reminders removida do RadioPlayer (~200 linhas)
- TODO futuro: reimplementar reminders se publico pedir

### 3. Validacao de cache de artwork

Adicionada verificacao de tamanho de ficheiro (> 0 bytes) no artworkCache para evitar entradas corrompidas por downloads parciais. Ficheiros truncados sao removidos automaticamente.

## Auditoria de Background (checkup)

Auditoria completa do sistema de background audio + metadata realizada. Conclusoes:

**Confirmado funcional:**
- expo-audio AudioControlsService NAO usa Glide para artwork do lock screen — usa `URL.openConnection()` direto, sem lifecycle issues
- `file://` URIs funcionam corretamente com `URL.openConnection()` no Android
- `artworkUrl=undefined` forca rebuild da notificacao via elvis operator no Kotlin
- Foreground service com `foregroundServiceType="mediaPlayback"` funciona em Android 14+
- SSE + polling continuam em background (JS thread mantida viva pelo foreground service)

**Corrigido nesta release:**
- Cache de artwork podia retornar ficheiros corrompidos (0 bytes) de downloads parciais

**Edge cases aceites:**
- Se duas musicas consecutivas usam o mesmo artwork URL, o bitmap nao e recarregado (correto para mesmo album)
- Se download de artwork falha, notification mostra artwork anterior (degradacao aceitavel)
