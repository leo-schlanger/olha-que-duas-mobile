# Release v1.11.0 (Build 48)

**Data**: 2026-04-17
**versionName**: 1.11.0
**versionCode**: 48 (Android)

---

## Resumo da Release

**Upgrade de Expo SDK 54 → 55** — o mais significativo salto de plataforma desde o lançamento da app.

Motivação principal: o `expo-audio@1.1.1` (SDK 54) tinha limitações hard-coded que não podiam ser contornadas:
- Sem preloading nativo → audio startup de 2-3s
- Bug de background (#38317) → áudio parava após ~3min em alguns devices
- Artwork loading nativo sem cache/timeout

SDK 55 traz `expo-audio@55.0.13` com **preload nativo** (PR #43061), bug fixes de background, e melhorias na MediaSession.

## Nova API: `preload()` do expo-audio@55

```typescript
import { preload, createAudioPlayer } from 'expo-audio';

// Pre-buffer the stream in background — data ready before user taps play
await preload({ uri: streamUrl });

// Later, when user taps play:
const player = createAudioPlayer({ uri: streamUrl });
player.play(); // Near-instant start — data already buffered
```

**Impacto**: audio startup passa de ~2-3s (ExoPlayer cold buffer) para near-instant quando preload é utilizado.

## expo-av removido

`expo-av@16.0.8` estava no package.json mas **não era importado em nenhum ficheiro src/**. Era um resíduo da migração para `expo-audio`. SDK 55 remove o expo-av oficialmente — nós já não o usávamos.
