# Release v1.11.5 (Build 53)

**Data**: 2026-04-19
**versionName**: 1.11.5
**versionCode**: 53 (Android)

---

## Resumo da Release

Correcao de dois bugs criticos descobertos em testes da v1.11.4:
1. Radio parava instantaneamente ao minimizar/adormecer ecra
2. Artwork do lock screen nao atualizava quando a musica trocava

---

## Correcoes

### 1. Grace period na deteccao de pause externo

**Ficheiro**: `src/services/radioService.ts`

**Problema**: Quando o app vai para background, o player nativo do Android reporta brevemente `playing=false` durante a transicao do Activity lifecycle. O codigo interpretava isto como "pause do lock screen" e parava TUDO — `isIntentionallyStopped = true`, unsubscribe do now-playing, emitia status parado.

**Solucao**: Adicionado `backgroundTransitionAt` timestamp. Durante 3 segundos apos a transicao para background, a deteccao de pause externo e suprimida. O player nativo estabiliza neste periodo e volta a reportar `playing=true`. Passados os 3s, a deteccao funciona normalmente para pause real do lock screen.

Proteccao aplicada nos dois caminhos:
- `handlePlaybackStatus()` (event-driven, dispara instantaneamente)
- `pollPlayerStatus()` (polling cada 5s)

### 2. Artwork do lock screen so via file:// URI

**Ficheiro**: `src/services/radioService.ts`

**Problema**: O `updateLockScreenMetadata()` recebia URLs remotas (`https://...`) quando o artwork nao estava cached. O Android tentava descarregar via `URL.openConnection()` que falha silenciosamente em background — titulo/artista atualizavam mas a foto ficava presa na musica anterior.

**Solucao**: `pickArt()` agora so retorna `file://` URIs:
- Se o artwork esta cached: retorna `file://cached-cover.jpg` (load <10ms)
- Se nao esta cached: retorna `file://cached-logo.jpg` (logo da radio)
- Quando o download async completa, `nowPlayingService` re-emite com `localArtUri` preenchido, triggerando segundo `updateLockScreenMetadata` com o `file://` correto

Resultado: em vez de foto presa na musica anterior, o user ve o logo brevemente (~1-3s) e depois a capa correta aparece.

### 3. Nova constante

**Ficheiro**: `src/config/constants.ts`

- `RADIO_BG_GRACE_PERIOD: 3000` — grace period para suprimir falsos pauses durante transicao para background
