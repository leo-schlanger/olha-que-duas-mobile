# v1.16.2 — Artwork sync + background reliability

**Build 69 | 2026-04-28**

## Problemas raiz identificados e corrigidos

### 1. logoUri ficava preso como URL remota (causa: prefetchLogo race)
- `radioService.logoUri` era resolvido UMA vez no primeiro `play()` via `getLogoUri()`
- Se `prefetchLogo()` (fire-and-forget) não tivesse terminado, retornava URL remota
- O campo ficava preso como URL remota para TODA a sessão — todas as notificações usavam HTTP
- **Fix:** `logoUri` agora é um getter que chama `getLogoUri()` a cada uso — automaticamente retorna `file://` assim que o prefetch completa

### 2. External pause detection matava playback em transições breves
- `handlePlaybackStatus()` e `pollPlayerStatus()` detectavam "external pause" quando o player emitia um status transitório `{playing:false, buffering:false}` durante hiccups de rede de 100ms
- Em foreground sem grace period, isto marcava `isIntentionallyStopped=true` e parava tudo
- **Fix:** Debounce de 500ms — se `lastPlayingAt` foi há menos de 500ms, o "pause" é ignorado como transitório

### 3. flushReady timeout Runnable nunca cancelado
- `mainHandler.postDelayed(flushReady, 10_000)` disparava mesmo que artwork chegasse em 2s
- Embora fosse no-op (callback já null), era um leak de Runnable
- **Fix:** `flushReadyTimeout` é um Runnable nomeado, cancelado com `removeCallbacks()` quando `flushReady()` executa

### 4. Payload ordering podia rejeitar payloads válidos após clock skew
- `lastProcessedPlayedAt` rejeitava qualquer payload com `played_at < last` sem timeout
- Se servidor AzuraCast reiniciasse ou ajustasse relógio, TODOS os payloads subsequentes seriam rejeitados
- **Fix:** Safety valve — só rejeita se o gap é menor que 30s. Após 30s sem payload processado, aceita o próximo

### 5. imageKey não incluía URI do artwork — Image não remontava ao trocar localArtUri
- `imageKey` era `título + artista + fgCount` — quando `localArtUri` mudava de `null` para `file://`, a key era idêntica
- expo-image podia servir bitmap stale do cache interno sem recarregar
- **Fix:** `imageKey` agora usa a URI final (`localArtUri || song.art`) em vez de título+artista — qualquer mudança de URI força remount
