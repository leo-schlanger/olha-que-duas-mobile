# Release v1.11.7 (Build 55)

**Data**: 2026-04-19
**versionName**: 1.11.7
**versionCode**: 55 (Android)

---

## Resumo da Release

Correcao da atualizacao de artwork (foto da musica) no componente in-app ao regressar do background. A foto ficava presa na musica anterior ate o utilizador interagir com o app.

---

## Correcoes

### 1. Forced re-render ao regressar do foreground

**Ficheiro**: `src/hooks/useNowPlaying.ts`

**Problema**: Quando o app ia para background, o `nowPlayingService` continuava a detetar trocas de musica (via SSE/polling) e a chamar `setData()` no hook. Porem, o React Native nao faz flush de renders enquanto o app esta invisivel. Ao voltar ao foreground, o componente podia nao re-renderizar com os dados mais recentes, deixando a foto antiga visivel.

**Solucao**: Adicionado listener de `AppState` dentro do `useEffect` do hook. Quando o estado muda para `'active'`, le `nowPlayingService.getCurrentData()` e chama `setData({ ...current })` com uma nova referencia de objeto. Isto garante que:
- O React deteta a mudanca de state (nova referencia)
- O `React.memo` do NowPlaying deteta a mudanca de props
- O expo-image recebe a nova source URI e carrega a imagem correta

### 2. recyclingKey baseado na identidade da musica

**Ficheiro**: `src/components/radio/NowPlaying.tsx`

**Problema**: O expo-image com `cachePolicy="memory-disk"` podia manter a imagem antiga no cache visual nativo, mesmo quando a source URI mudava apos retorno do background.

**Solucao**: Adicionado `recyclingKey` a todos os componentes `<Image>` de artwork, baseado na **identidade do conteudo** (nao na URI):

| Modo | recyclingKey |
|---|---|
| `music` | `${song.title}\0${song.artist}` |
| `podcast` | `podcastName` |
| `announcement` | `announcementName` |

A chave so muda quando a musica/conteudo realmente muda. Quando apenas a URI muda para a mesma musica (remote URL тЖТ cached file://), o `recyclingKey` permanece estavel тАФ o expo-image faz swap suave sem flash em branco.

---

## Comportamento Garantido

### Foreground (app ativo)
1. SSE/Polling deteta nova musica тЖТ `emit()` тЖТ `setData()` тЖТ React re-renderiza
2. `recyclingKey` muda (nova musica) тЖТ expo-image reseta view nativa
3. `transition={350}` тЖТ fade-in da nova artwork
4. Download async completa тЖТ re-emit com `localArtUri` тЖТ `recyclingKey` estavel тЖТ swap suave sem flash

### Background тЖТ Foreground
1. App em background: JS thread ativa, SSE/polling continuam
2. Musica muda: `emit()` тЖТ `setData()` enfileirado, React nao renderiza
3. Artwork pre-baixada para cache local (file://)
4. User volta ao app тЖТ `AppState` тЖТ `'active'`
5. Hook le `getCurrentData()` тЖТ `setData({ ...current })` тЖТ re-render forcado
6. `recyclingKey` diferente тЖТ expo-image carrega nova foto (<10ms de file://)
