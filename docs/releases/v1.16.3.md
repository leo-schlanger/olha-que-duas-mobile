# v1.16.3 — Critical background fix

**Build 70 | 2026-04-28**

## Bug crítico corrigido

### Background playback morria imediatamente
- **Causa raiz:** Na v1.16.1, adicionei `MediaService.instance?.stopSelf()` no `OnDestroy` do `ExpoMediaSessionModule` para "prevenir serviço órfão"
- **Problema:** O `OnDestroy` do módulo Expo dispara quando a **Activity** é destruída pelo Android para liberar memória — o que acontece NORMALMENTE quando o app vai para background. Isto matava o foreground service, que matava o audio
- **Fix:** Removido `stopSelf()` do `OnDestroy`. O service só para via `deactivate()` (chamado explicitamente pelo JS) ou `onTaskRemoved` (user swipe dos recents)

### lastPlayingAt inicializado em play()
- O debounce de external pause (500ms) podia falhar logo após play() porque `lastPlayingAt` era 0
- Agora `lastPlayingAt = Date.now()` é definido no início de `play()`, dando 500ms de margem para o player transicionar para playing
