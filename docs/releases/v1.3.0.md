**Data**: 2026-04-08

## Resumo da Release

Release focada em robustez, acessibilidade e correcao de bugs. Inclui finalizacao do i18n, correcao de bugs de radio em background, melhoria de layout da programacao diaria, e 26 correcoes identificadas numa auditoria completa de 5 pilares (resiliencia, performance, seguranca, UX/a11y, lifecycle).

---

## Alteracoes

### Correcoes de Bugs

- **Radio background**: Notificacao do lock screen ja nao fica orfao ao fechar a app. Delay de 100ms entre desativacao do lock screen e release do player
- **Radio init**: `isInitialized` ja nao e marcado `true` quando inicializacao falha, permitindo retry automatico
- **Race condition na init**: Settings carregados uma unica vez e passados ao radioService (evita double-load)
- **Timeout de inicializacao**: App nao fica presa no splash - timeout de 8s com banner de retry
- **Crash null content**: `post.content.split()` protegido contra null/undefined
- **Layout programacao diaria**: Grid 2x2 substituido por layout 1-por-linha para melhor legibilidade

### Internacionalizacao (i18n) - Finalizada

- Ultimas strings hardcoded traduzidas: NewsDetailScreen (4 strings), DailyScheduleSection ("AGORA")
- Novas chaves: `news.notFound`, `news.back`, `news.viewOriginalSource`, `news.toggleFilters`, `radio.dailySchedule.now`, `consent.mustChoose`

### Acessibilidade

- Touch targets aumentados para 44x44 minimo: filter toggle, info button, close button, refresh button
- Accessibility labels adicionados em botoes criticos: refresh (RadioControls, WeatherScreen), filter toggle (NewsScreen), close (AboutBottomSheet)
- `prefers-reduced-motion` respeitado no RadioVisualizer (barras estaticas quando ativado)

### Performance

- Polling de rede: intervalo fixo 10s (era 3s offline, causava drain de bateria)
- Ad SDK polling: 500ms -> 1000ms
- `fetchWithRetry`: nova funcao com backoff exponencial para chamadas de rede
- Guards `isMounted` em useRadio e useWeather para evitar memory leaks

### Resiliencia

- `ScreenErrorBoundary`: Novo componente para isolar erros por screen (uma screen crashing nao derruba a app toda)
- `initState === 'failed'`: Mostra banner de erro com botao retry em vez de ignorar silenciosamente
- GDPR modal: Back button Android mostra toast explicativo em vez de nao fazer nada
- BackHandler: Listener adequado com `useFocusEffect` no RadioScreen

### Seguranca

- `safeOpenUrl`: Nova utilidade que valida protocolo (https/http/mailto) antes de abrir URLs externas
- Tech debt documentado: Premium/IAP sem validacao server-side

### Outras Melhorias

- `currentDay`/`currentPeriod` atualizam ao voltar do background (resolve exibicao errada apos meia-noite)
- `KeyboardAvoidingView` no NewsScreen para search input
- `OfflineBanner` usa cores do tema em vez de hardcoded
- Deep link `exact: true` no Radio tab
- `AsyncStorage.setItem` awaited no PremiumContext
